---
title: "Get Current User"
api: "GET /api/v1/auth/me"
---

Retrieve the currently authenticated user's information including their profile, roles, and permissions.

## Authentication

This endpoint requires a valid Bearer token in the Authorization header.

```bash
Authorization: Bearer YOUR_TOKEN_HERE
```

## Response

<ResponseField name="user" type="object">
  The authenticated user object
  
  <Expandable title="user properties">
    <ResponseField name="id" type="integer">
      User's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      User's full name
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="email_verified_at" type="string" nullable>
      Timestamp when email was verified
    </ResponseField>
    
    <ResponseField name="active" type="boolean">
      Whether the user account is active
    </ResponseField>
    
    <ResponseField name="last_login_at" type="string" nullable>
      Timestamp of last successful login
    </ResponseField>
    
    <ResponseField name="last_login_ip" type="string" nullable>
      IP address of last login
    </ResponseField>
    
    <ResponseField name="failed_login_attempts" type="integer">
      Number of consecutive failed login attempts
    </ResponseField>
    
    <ResponseField name="locked_until" type="string" nullable>
      Timestamp until which account is locked (if locked)
    </ResponseField>
    
    <ResponseField name="roles" type="array">
      Array of role names assigned to the user (e.g., ["super_admin", "admin"])
    </ResponseField>
    
    <ResponseField name="permissions" type="array">
      Array of permission names granted to the user
    </ResponseField>
  </Expandable>
</ResponseField>

## Code Examples

<CodeGroup>

```bash cURL
curl -X GET https://your-domain.com/api/v1/auth/me \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Accept: application/json"
```

```javascript JavaScript
const response = await fetch('https://your-domain.com/api/v1/auth/me', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN_HERE',
    'Accept': 'application/json'
  }
});

const data = await response.json();
console.log(data.user);
```

```php PHP
$ch = curl_init();
curl_setopt($ch, CURLOPT_URL, 'https://your-domain.com/api/v1/auth/me');
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Authorization: Bearer YOUR_TOKEN_HERE',
    'Accept: application/json'
]);

$response = curl_exec($ch);
$user = json_decode($response, true);
curl_close($ch);
```

</CodeGroup>

## Error Responses

### 401 Unauthorized

Returned when the token is missing, invalid, or expired.

```json
{
  "message": "Unauthenticated."
}
```

## Use Cases

- **Profile Display**: Show current user information in the application UI
- **Permission Checking**: Determine what actions the user can perform
- **Session Validation**: Verify that the user's token is still valid
- **Role-Based UI**: Conditionally render UI elements based on user roles
