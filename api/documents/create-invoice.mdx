---
title: Create Invoice
api: POST /api/v1/invoices
---

Creates a new electronic invoice (Factura) for SUNAT. The invoice is created in draft status and must be sent to SUNAT separately.

<Note>
Facturas are issued to business customers (RUC) and support advanced features like credit terms, detraction, perception, and retention.
</Note>

## Authentication

This endpoint requires authentication using a Bearer token (Sanctum).

```bash
Authorization: Bearer YOUR_API_TOKEN
```

## Request Body

### Company and Branch

<ParamField body="company_id" type="integer" required>
  ID of the company issuing the invoice. Must exist in the companies table.
</ParamField>

<ParamField body="branch_id" type="integer" required>
  ID of the branch issuing the invoice. Must belong to the specified company.
</ParamField>

<ParamField body="serie" type="string" required>
  Invoice series code (max 4 characters). Must start with 'F' for facturas (e.g., "F001").
</ParamField>

### Dates and Currency

<ParamField body="fecha_emision" type="date" required>
  Invoice issue date in YYYY-MM-DD format.
</ParamField>

<ParamField body="fecha_vencimiento" type="date">
  Due date for the invoice. Must be equal to or after the issue date. Required for credit invoices.
</ParamField>

<ParamField body="moneda" type="string" required>
  Currency code. Allowed values: `PEN` (Peruvian Sol), `USD` (US Dollar).
</ParamField>

<ParamField body="tipo_operacion" type="string">
  Type of operation code (max 4 characters).
  - `0101`: Internal sale (default)
  - `0200`: Export
  - `0112`: Sale with IVAP
  - Other SUNAT catalog 51 codes
</ParamField>

### Payment Terms

<ParamField body="forma_pago_tipo" type="string" required>
  Payment type. Allowed values: `Contado` (cash), `Credito` (credit).
</ParamField>

<ParamField body="forma_pago_cuotas" type="array">
  Payment installments. Required when `forma_pago_tipo` is `Credito`.
  
  <Expandable title="Installment object">
    <ParamField body="forma_pago_cuotas[].moneda" type="string" required>
      Currency of the installment.
    </ParamField>
    
    <ParamField body="forma_pago_cuotas[].monto" type="number" required>
      Installment amount (min: 0).
    </ParamField>
    
    <ParamField body="forma_pago_cuotas[].fecha_pago" type="date" required>
      Installment payment date.
    </ParamField>
  </Expandable>
</ParamField>

### Client Information

<ParamField body="client" type="object" required>
  Customer information.
  
  <Expandable title="Client object">
    <ParamField body="client.tipo_documento" type="string" required>
      Document type code:
      - `1`: DNI (National ID)
      - `4`: Carnet de extranjería
      - `6`: RUC (Tax ID)
      - `0`: Other
    </ParamField>
    
    <ParamField body="client.numero_documento" type="string" required>
      Document number (max 15 characters).
    </ParamField>
    
    <ParamField body="client.razon_social" type="string" required>
      Legal name or business name (max 255 characters).
    </ParamField>
    
    <ParamField body="client.nombre_comercial" type="string">
      Trade name (max 255 characters).
    </ParamField>
    
    <ParamField body="client.direccion" type="string">
      Address (max 255 characters).
    </ParamField>
    
    <ParamField body="client.ubigeo" type="string">
      UBIGEO code (6 digits). Geographic location code.
    </ParamField>
    
    <ParamField body="client.distrito" type="string">
      District name (max 100 characters).
    </ParamField>
    
    <ParamField body="client.provincia" type="string">
      Province name (max 100 characters).
    </ParamField>
    
    <ParamField body="client.departamento" type="string">
      Department name (max 100 characters).
    </ParamField>
    
    <ParamField body="client.telefono" type="string">
      Phone number (max 20 characters).
    </ParamField>
    
    <ParamField body="client.email" type="email">
      Email address (max 100 characters).
    </ParamField>
  </Expandable>
</ParamField>

### Invoice Items

<ParamField body="detalles" type="array" required>
  Invoice line items (minimum 1 item required).
  
  <Expandable title="Item object">
    <ParamField body="detalles[].codigo" type="string" required>
      Product/service code (max 50 characters).
    </ParamField>
    
    <ParamField body="detalles[].descripcion" type="string" required>
      Item description (max 500 characters).
    </ParamField>
    
    <ParamField body="detalles[].unidad" type="string" required>
      Unit of measure code (max 3 characters). Examples: `NIU` (unit), `ZZ` (service).
    </ParamField>
    
    <ParamField body="detalles[].cantidad" type="number" required>
      Quantity (min: 0.001).
    </ParamField>
    
    <ParamField body="detalles[].mto_valor_unitario" type="number" required>
      Unit value excluding taxes (min: 0).
    </ParamField>
    
    <ParamField body="detalles[].porcentaje_igv" type="number">
      IGV (VAT) percentage. Typically 18. Not used for exports.
    </ParamField>
    
    <ParamField body="detalles[].porcentaje_ivap" type="number">
      IVAP percentage for rice sales (0-100).
    </ParamField>
    
    <ParamField body="detalles[].mto_valor_gratuito" type="number">
      Free item value (min: 0). For promotional items.
    </ParamField>
    
    <ParamField body="detalles[].tip_afe_igv" type="string">
      IGV affectation type code:
      - `10`: Taxable - Onerosa Operation
      - `11`: Taxable - Free Withdrawal
      - `12`: Taxable - Withdrawal by Prize
      - `13`: Taxable - Donation
      - `14`: Taxable - Promotion
      - `15`: Taxable - Consumed Himself
      - `16`: Taxable - Other Free
      - `17`: Taxable - IVAP
      - `20`: Exempt - IGV/IVAP Transfer
      - `21`: Exempt - Free Transfer
      - `30`: Non-Taxable - Onerosa Operation
      - `31`: Non-Taxable - Free Withdrawal
      - `32`: Non-Taxable - Withdrawal by Prize
      - `33`: Non-Taxable - Donation
      - `34`: Non-Taxable - Others
      - `35`: Non-Taxable - Sale Service
      - `36`: Non-Taxable - Not Domiciled
      - `40`: Export
    </ParamField>
    
    <ParamField body="detalles[].codigo_producto_sunat" type="string">
      SUNAT product code (max 50 characters).
    </ParamField>
  </Expandable>
</ParamField>

## Tax Calculation

The API automatically calculates taxes based on the item details:

- **Base IGV**: Calculated from `mto_valor_unitario` × `cantidad` for taxable items
- **IGV Amount**: Base × (`porcentaje_igv` / 100)
- **Unit Price**: (Value + IGV) / quantity
- **Export Operations** (`tipo_operacion: 0200`): IGV is set to 0, and `tip_afe_igv` is set to `40`
