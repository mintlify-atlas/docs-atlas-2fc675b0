---
title: Create Boleta
api: POST /api/v1/boletas
---

Creates a new electronic sales receipt (Boleta de Venta) for SUNAT. Boletas are issued to individual customers and can be sent individually or via daily summary.

<Note>
Boletas are issued to individual consumers (DNI) and have simpler requirements than invoices. They can be sent to SUNAT immediately or grouped in a daily summary.
</Note>

## Authentication

This endpoint requires authentication using a Bearer token (Sanctum).

```bash
Authorization: Bearer YOUR_API_TOKEN
```

## Request Body

### Company and Branch

<ParamField body="company_id" type="integer" required>
  ID of the company issuing the boleta. Must exist in the companies table.
</ParamField>

<ParamField body="branch_id" type="integer" required>
  ID of the branch issuing the boleta. Must belong to the specified company.
</ParamField>

<ParamField body="serie" type="string" required>
  Boleta series code (max 4 characters). Must start with 'B' for boletas (e.g., "B001").
</ParamField>

### Dates and Configuration

<ParamField body="fecha_emision" type="date" required>
  Boleta issue date in YYYY-MM-DD format.
</ParamField>

<ParamField body="ubl_version" type="string">
  UBL version (max 5 characters). Default: "2.1".
</ParamField>

<ParamField body="tipo_operacion" type="string">
  Type of operation code (max 4 characters). Default: "0101" (internal sale).
</ParamField>

<ParamField body="moneda" type="string">
  Currency code (max 3 characters). Default: "PEN".
</ParamField>

### Sending Method

<ParamField body="metodo_envio" type="string" required>
  How the boleta will be sent to SUNAT:
  - `individual`: Send immediately to SUNAT (for boletas > 700 PEN or to foreign customers)
  - `resumen_diario`: Include in daily summary (for boletas ≤ 700 PEN to local customers)
</ParamField>

<Warning>
According to SUNAT regulations, boletas over 700 PEN or issued to foreign customers must be sent individually, not in daily summaries.
</Warning>

### Payment Terms (Optional)

<ParamField body="forma_pago_tipo" type="string">
  Payment type (max 20 characters). Example: "Contado", "Credito".
</ParamField>

<ParamField body="forma_pago_cuotas" type="array">
  Payment installments if applicable.
</ParamField>

### Client Information

<ParamField body="client" type="object" required>
  Customer information.
  
  <Expandable title="Client object">
    <ParamField body="client.tipo_documento" type="string" required>
      Document type code (max 1 character):
      - `1`: DNI (most common for boletas)
      - `4`: Carnet de extranjería
      - `6`: RUC (for businesses)
      - `0`: Other
    </ParamField>
    
    <ParamField body="client.numero_documento" type="string" required>
      Document number (max 15 characters).
    </ParamField>
    
    <ParamField body="client.razon_social" type="string" required>
      Customer name or business name (max 255 characters).
    </ParamField>
    
    <ParamField body="client.nombre_comercial" type="string">
      Trade name (max 255 characters).
    </ParamField>
    
    <ParamField body="client.direccion" type="string">
      Address (max 255 characters).
    </ParamField>
    
    <ParamField body="client.ubigeo" type="string">
      UBIGEO code (max 6 characters). Geographic location code.
    </ParamField>
    
    <ParamField body="client.distrito" type="string">
      District name (max 100 characters).
    </ParamField>
    
    <ParamField body="client.provincia" type="string">
      Province name (max 100 characters).
    </ParamField>
    
    <ParamField body="client.departamento" type="string">
      Department name (max 100 characters).
    </ParamField>
    
    <ParamField body="client.telefono" type="string">
      Phone number (max 20 characters).
    </ParamField>
    
    <ParamField body="client.email" type="email">
      Email address (max 255 characters).
    </ParamField>
  </Expandable>
</ParamField>

### Boleta Items

<ParamField body="detalles" type="array" required>
  Boleta line items (minimum 1 item required).
  
  <Expandable title="Item object">
    <ParamField body="detalles[].codigo" type="string" required>
      Product/service code (max 30 characters).
    </ParamField>
    
    <ParamField body="detalles[].descripcion" type="string" required>
      Item description (max 255 characters).
    </ParamField>
    
    <ParamField body="detalles[].unidad" type="string" required>
      Unit of measure code (max 3 characters). Examples: `NIU` (unit), `ZZ` (service).
    </ParamField>
    
    <ParamField body="detalles[].cantidad" type="number" required>
      Quantity (min: 0.01).
    </ParamField>
    
    <ParamField body="detalles[].mto_valor_unitario" type="number" required>
      Unit value excluding taxes (min: 0).
    </ParamField>
    
    <ParamField body="detalles[].mto_valor_gratuito" type="number">
      Free item value (min: 0). For promotional items.
    </ParamField>
    
    <ParamField body="detalles[].porcentaje_igv" type="number" required>
      IGV (VAT) percentage (0-100). Typically 18.
    </ParamField>
    
    <ParamField body="detalles[].porcentaje_ivap" type="number">
      IVAP percentage for rice sales (0-100).
    </ParamField>
    
    <ParamField body="detalles[].tip_afe_igv" type="string" required>
      IGV affectation type code (max 2 characters):
      - `10`: Taxable - Onerosa Operation (most common)
      - `11`: Taxable - Free Withdrawal
      - `20`: Exempt
      - `30`: Non-taxable
      - `40`: Export
    </ParamField>
    
    <ParamField body="detalles[].isc" type="number">
      ISC (Selective Consumption Tax) amount (min: 0).
    </ParamField>
    
    <ParamField body="detalles[].icbper" type="number">
      ICBPER (Plastic Bag Tax) amount (min: 0).
    </ParamField>
    
    <ParamField body="detalles[].factor_icbper" type="number">
      ICBPER factor/quantity for calculation (min: 0).
    </ParamField>
  </Expandable>
</ParamField>

<Warning>
The boleta is created in `PENDIENTE` (pending) status. Depending on the `metodo_envio`:
- **individual**: Call [Send to SUNAT](/api/documents/send-to-sunat) immediately
- **resumen_diario**: Include it in the daily summary at the end of the day
</Warning>

## Sending Methods Explained

### Individual Sending

Use `metodo_envio: "individual"` when:
- Boleta amount > 700 PEN
- Customer is a foreign national
- Immediate SUNAT validation is required

After creation, call `POST /api/v1/boletas/{id}/send-sunat` to submit to SUNAT.

### Daily Summary

Use `metodo_envio: "resumen_diario"` when:
- Boleta amount ≤ 700 PEN
- Customer is a local individual (DNI)
- Batch processing is acceptable

Boletas will be grouped and sent in a single daily summary document at the end of the business day.
